## Font Pair Guide

All fonts are from Google Fonts. Use [Google Fonts Helper](https://gwfh.mranftl.com/fonts) to subset and host locally.

---

### 1. Roboto & Nunito
**Suggested Industries:** Medical, Health, SaaS, Tech

- **Roboto** (headings) — Mechanical skeleton with geometric forms but friendly open curves. Natural reading rhythm similar to humanist and serif types.
- **Nunito** (body) — Well-balanced rounded sans-serif superfamily. Approachable and clean.

---

### 2. Syne & Inter
**Suggested Industries:** Food, Health + Care, Youth, Beauty

- **Syne** (headings) — Originally designed for an Art Center. Geometric sans-serif that gets wider as it gets bolder. Atypical and expressive.
- **Inter** (body) — Variable font designed for screens. Tall x-height for readability. Includes contextual alternates, slashed zero, tabular numbers.

---

### 3. Montserrat & Lato
**Suggested Industries:** Landscape, Home Improvement, Trades, General Use

- **Montserrat** (headings) — Based on urban typography from the Montserrat neighborhood of Buenos Aires. Classic proportions with modern feel.
- **Lato** (body) — "Transparent" in body text, distinctive at large sizes. Classical proportions with a sleek sans-serif finish.

---

### 4. Noto Serif & Noto Sans
**Suggested Industries:** Construction, Landscape, Apparel, Higher-End Designs

- **Noto Serif** (headings) — Part of Google's global font collection. Modulated serif design, supports 3,256 glyphs.
- **Noto Sans** (body) — Unmodulated sans-serif counterpart. Clean and highly legible.

---

### 5. Playfair Display & DM Sans
**Suggested Industries:** Food, Health + Care, Home Improvement, Fancier Designs

- **Playfair Display** (headings) — Transitional serif influenced by John Baskerville and Scotch Roman. High contrast, delicate hairlines. Elegant and editorial.
- **DM Sans** (body) — Low-contrast geometric sans-serif designed for smaller text sizes. Clean and modern.

---

### 6. Oswald & Source Serif Pro
**Suggested Industries:** Tech, Construction, Landscape, Beauty

- **Oswald** (headings) — Reworked from classic Alternate Gothic. Condensed, strong, and web-optimized.
- **Source Serif Pro** (body) — Transitional serif designed to complement sans-serifs. Loosely based on Pierre Simon Fournier. Pairs naturally with Oswald.

---

### 7. Stint Ultra Expanded & Pontano Sans
**Suggested Industries:** Health, Landscape, Home Improvement

- **Stint Ultra Expanded** (headings) — Ultra-wide square serif. Powerful and bold for headlines where space is not a concern.
- **Pontano Sans** (body) — Minimalist, light-weighted sans-serif. Works as display or body text.

---

### 8. Space Grotesk & Inconsolata
**Suggested Industries:** Food, Health, Youth, Tech

- **Space Grotesk** (headings) — Proportional sans-serif based on Space Mono. Retains monospace idiosyncrasies while optimizing for readability.
- **Inconsolata** (body) — Monospace font with attention to detail for high-resolution rendering. Unusual pairing for tech-forward brands.

---

### 9. Caudex & Merriweather
**Suggested Industries:** Beauty, Luxury, Natural

- **Caudex** (headings) — Classic origins from the late 90s. Includes Medieval Unicode support. Elegant and refined.
- **Merriweather** (body) — Designed for screen readability. Large x-height, mild diagonal stress, sturdy serifs, and open forms.

---

### 10. Poppins & Lora
**Suggested Industries:** SaaS, Tech, Medical, Home Improvement, General Use

- **Poppins** (headings) — Geometric sans-serif with international support (Latin + Devanagari). Clean, modern, widely used.
- **Lora** (body) — Contemporary serif rooted in calligraphy. Brushed curves contrasted with driving serifs. Warm and memorable.

## Page Speed Handbook

A step-by-step guide to achieving 100/100 Google PageSpeed scores.

---

### Step 1 — Optimizing Images

#### Use the `<picture>` Element

Always use `<picture>` instead of a plain `<img>` tag. This lets you serve different image sizes for mobile vs desktop.

```html
<picture>
  <!-- Mobile Image -->
  <source media="(max-width: 600px)" srcset="/images/hero-m.webp">
  <!-- Desktop Image -->
  <source media="(min-width: 601px)" srcset="/images/hero.webp">
  <!-- Fallback -->
  <img aria-hidden="true" loading="lazy" decoding="async"
       src="/images/hero.webp" alt="description" width="1920" height="1280">
</picture>
```

#### Crop Images to Display Size

Never serve an image larger than it appears on screen. Find the display size:
- Hover over the image in browser dev tools to see displayed dimensions
- Crop to **2x the display size** (to account for high-density screens)
- Save mobile crops with a `-m` suffix (e.g. `hero-m.webp`)

**Example:** A 4760×3173px (1MB) image cropped to 1102×500px drops to 108KB — a 90% reduction.

#### Compress Images

Use [Compressor.io](https://compressor.io/) to bulk compress.

Target sizes:
- Mobile images: **20–40KB**
- Background images: **never over 100KB**

#### Convert to WEBP

Use [CloudConvert](https://cloudconvert.com/jpg-to-webp) to convert to WEBP format.

- WEBP is ~30% smaller than JPG
- WEBP is ~26% smaller than PNG
- Translates to ~25–35% faster load times

#### Lazy Load All Below-the-Fold Images

Add `loading="lazy"` to every image that is not visible on initial page load.

```html
<img loading="lazy" decoding="async" src="..." alt="..." width="..." height="...">
```

**Never lazy load images above the fold** — this causes CLS (Content Layout Shift).

#### Add `decoding="async"` to Every Image

```html
<img decoding="async" ...>
```

This tells the browser to decode the image later so other content displays first.

#### Optimize Landing Page Background Images

On mobile, browsers crop the left and right of wide background images anyway. Instead of loading the full-width image on mobile:
- Create a center-cropped version sized to phone screen width
- Use the `<picture>` element to serve the right size per screen

#### Preload Critical Above-the-Fold Images

Add preload links in the `<head>` for images visible on page load (hero image, logo):

```html
<link rel="preload" as="image" href="/images/hero-m.webp">
```

#### Lazy Load Background Images in CSS (via HTML)

Instead of `background-image` in CSS, use a `<picture>` element inside the section:

```html
<section id="cta">
  <div class="container">...</div>
  <picture>
    <source media="(max-width: 600px)" srcset="/images/bg-m.webp">
    <source media="(min-width: 601px)" srcset="/images/bg.webp">
    <img aria-hidden="true" loading="lazy" decoding="async"
         src="/images/bg.webp" alt="" width="1920" height="1280">
  </picture>
</section>
```

```css
#cta {
  position: relative;
  z-index: 1;
}

#cta picture {
  position: absolute;
  top: 0;
  left: 0;
  width: 100%;
  height: 100%;
  z-index: -2;
}

#cta picture img {
  position: absolute;
  top: 0;
  left: 0;
  width: 100%;
  height: 100%;
  object-fit: cover;
}
```

Benefits: lazy loadable, easier file path management, mobile/desktop images in one place.

### Step 2 — Optimizing Fonts

#### Do NOT Use Google Fonts CDN

The CDN link loads unnecessary characters and render blocks your page. Use local hosting instead.

#### Use Google Fonts Helper to Download Fonts

Tool: https://gwfh.mranftl.com/fonts

Steps:
1. Search for your font
2. Select **Latin** subset only (removes unused characters — reduces file size from ~180KB to ~18KB)
3. Select only the weights you need (regular, 700, italic at most — fewer = faster)
4. Click **Modern Browsers** to get the CSS
5. Download font files and place in `/assets/fonts/`
6. Paste the CSS into your root stylesheet
7. **Add `font-display: swap;`** to each `@font-face` declaration

```css
@font-face {
  font-family: 'Roboto';
  font-style: normal;
  font-weight: 400;
  font-display: swap;
  src: local(''),
       url('/assets/fonts/roboto-v30-latin-regular.woff2') format('woff2'),
       url('/assets/fonts/roboto-v30-latin-regular.woff') format('woff');
}
```

#### Preload Your Fonts

Add preload links in `<head>` for fonts used above the fold:

```html
<link rel="preload" href="/assets/fonts/roboto-v30-latin-regular.woff2"
      as="font" type="font/woff2" crossorigin>
```

Only preload the weights actually used on the landing section. This is often the difference between 98 and 100.

#### Fix Flash of Unstyled Content (FOUC / CLS)

When fonts swap in, text can reflow and cause CLS. Fix it by matching your web font to a system font fallback.

Tools:
- Font Style Matcher: https://meowni.ca/font-style-matcher/
- Browser default fonts reference: https://blog.hubspot.com/website/web-safe-html-css-fonts

Steps:
1. In the font matcher, set your web font on the right and a similar system font on the left
2. Match font size, line height, and letter spacing until they align
3. Use that fallback font in your CSS font-family stack
4. Fine-tune line-height in small increments (e.g. 1.49em → 1.51em) until CLS = 0

**Common cause of large CLS:** When the fallback font fits an extra word on a line that the web font bumps to the next line. Fix by slightly reducing font size so the word stays on the same line in both fonts.

### Step 3 — Everything Else

#### Google Analytics Placement

Place Google Analytics at the **bottom of `<head>`**, right above `<title>`. This prevents it from render blocking content above it.

#### Minify CSS and JS

Use Netlify's built-in asset optimization (enable in Build & Deploy settings). Uncheck "Bundle CSS" — it creates a large render-blocking bundle.

Alternatively, use free online minifiers.

#### Use SVGs Wherever Possible

- Download from [Flaticon](https://www.flaticon.com/) or [SVGRepo](https://www.svgrepo.com/)
- Serve in `<img>` tags (keeps DOM clean)
- Optimize all SVGs with: https://jakearchibald.github.io/svgomg/

#### Defer Non-Essential JavaScript

Place non-critical scripts at the bottom of `<body>` with the `defer` attribute:

```html
<script defer src="/assets/js/nav.js"></script>
```

#### Remove jQuery and Font Awesome

Both are unnecessary third-party CDN loads. Modern vanilla JS covers what jQuery did. Replace Font Awesome icons with downloaded SVGs from Flaticon.

#### Defer Non-Critical CSS (Render Blocking Fix)

When a large CSS file causes render blocking warnings:

1. Extract only the above-the-fold CSS into a `critical.css` file
2. Defer the main stylesheet:

```html
<link rel="stylesheet" href="/css/critical.css">
<link rel="stylesheet" href="/css/local.css"
      media="print" onload="this.media='all'; this.onload=null;">
<noscript>
  <link rel="stylesheet" href="/css/local.css">
</noscript>
```

- `critical.css` loads immediately (above-the-fold styles only)
- `local.css` defers until after page load
- `<noscript>` fallback loads `local.css` for browsers with JS disabled

This technique is often the final push from 98–99 to 100.

## Quick Reference Checklist

Use this before every site launch:

- [ ] All images use `<picture>` element with mobile + desktop sources
- [ ] All images cropped to 2x display size
- [ ] All images compressed via Compressor.io
- [ ] All images converted to WEBP
- [ ] All below-fold images have `loading="lazy"`
- [ ] All images have `decoding="async"`
- [ ] All images have `alt`, `width`, and `height` attributes
- [ ] Landing page image has a mobile center crop
- [ ] Critical above-fold images preloaded in `<head>`
- [ ] Background images served via `<picture>` in HTML (not CSS)
- [ ] Fonts downloaded and hosted locally via Google Fonts Helper
- [ ] Only Latin subset and necessary weights downloaded
- [ ] `font-display: swap` added to all `@font-face` declarations
- [ ] Fonts preloaded in `<head>`
- [ ] FOUC / CLS fixed with fallback font matching
- [ ] Google Analytics placed above `<title>` in `<head>`
- [ ] CSS and JS minified
- [ ] SVGs optimized via SVGOMG
- [ ] Non-critical JS deferred
- [ ] jQuery and Font Awesome removed
- [ ] Critical CSS separated and main CSS deferred
- [ ] HTML validated at validator.w3.org
- [ ] Links checked at drlinkcheck.com
- [ ] Lighthouse score 100 across all categories
